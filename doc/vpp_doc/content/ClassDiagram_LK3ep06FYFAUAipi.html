Model Elements
Name Description
  MTokenForest : Class

MTokenForest is a generic, persistence abstract super class.

It knows its state and value.

MTokenForest is obersevable (by UTokenForest).

  MGrid : Class

Grid is a Pure Fabrication class to manage forest tokens. It creates, knows and manages them.

It is responsible for the forest tokens creations, to face up a single token to get it value.

It has a reference to a strategy for choosing the next forest token and for shuffling the tokens.

  GameState : Class

This class is responsible for setting the state and getting what state we are in

  MHutToken : Class

Class representing hut tokens.

  MResource : Class

Is a persistent entity.

It knows its type.

  MStoneAgeGame : Class

MStoneAgeGame is a singleton facade controller.

It is responsible to play a single player turn, build a hut token, steal a resource from a player.

It knows board, grid, players, game state, game mode, game goal, the game strategy to get the next player.

  MPlayer : Class

The class Player is responsible for the marker and settlement creation, marker movement, build a hut token, steal a resource, choose the name player to rob.

It knows if is the winner.

  MMarker : Class

Persistent class.

It knows its name, in which square it is.

It is observable.

  MHumanPlayer : Class

A specific player.

  MEmulatedPlayer : Class

A specific player. The system emulates the human actions.

  MSettlement : Class

Java class to manage player's resources.

It is observable (by ResourceSquare and UPlayer).

  MBoard : Class

MBoard class is responsible to delegate the SquareFactory for the square creation.

MBoard class knows all board squares, how they are connected and which is the start square.

  MBuildingSiteSquare : Class

MBuildingSiteSquare is a persistence entity.

It also knows the action to perform if the marker lands on it. Indeed, it manage the action.

It is responsible for the creation of all hut tokens and it knows them and handle them until they are built.

  MRegularSquare : Class

It knows its type, the action to perform if the marker lands on it and how to setup the square.

  MSquare : Class

MSquare is a persistence entity.

It knows its name, its the next square with which it is linked, if it is the start square.

It knows the action to perform if the marker lands on it and how to setup the square.

It is the observable of the OBSERVER PATTERN.

  MResourceSquare : Class

It is responsible for the creation of all the Resource of the same type of the square.

Delegate the actual square state to perform the action if the marker's player lands on it.

It knows its type, all the resources on the square and how to setup the square..

It has a number of internal states and has a reference to the actual internal state.

It is the context of the STATE PATTERN.

It is the observable of the OBSERVER PATTERN (between MSquare and UResourceSquare).

It is the observer of the OBSERVER PATTER (between MSEttelment and MREsourceSquare).

  MSurpriseSquare : Class

It is responsible for the creation of all the Surprise Tokens.

It knows the action to perform if the marker lands on it and how to setup the square.

It knows its type.

  ISurpriseTokenCommand : Class

Interface for all the surprise token to perform a generic operation.
