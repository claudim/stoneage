Model Elements
Name Description
  MSquare : Class

MSquare is a persistence entity.

It knows its name, its the next square with which it is linked, if it is the start square.

It knows the action to perform if the marker lands on it and how to setup the square.

It is the observable of the OBSERVER PATTERN.

  ActionResult : Class

List of action result.

  MRegularSquare : Class

It knows its type, the action to perform if the marker lands on it and how to setup the square.

  MBoard : Class

MBoard class is responsible to delegate the SquareFactory for the square creation.

MBoard class knows all board squares, how they are connected and which is the start square.

  MRedBoard : Class
  MGreenBoard : Class
  MResourceSquare : Class

It is responsible for the creation of all the Resource of the same type of the square.

Delegate the actual square state to perform the action if the marker's player lands on it.

It knows its type, all the resources on the square and how to setup the square..

It has a number of internal states and has a reference to the actual internal state.

It is the context of the STATE PATTERN.

It is the observable of the OBSERVER PATTERN (between MSquare and UResourceSquare).

It is the observer of the OBSERVER PATTER (between MSEttelment and MREsourceSquare).

  ISquareState : Class

Interface to define all the behaviour of a resource square state.

  Release2ResourcesState : Class

It is a concrete state of the STATE PATTERN of the resource square.

Concrete state when the resource square has to release 2 resources.

It has a reference to the square.

  StealResourceState : Class

It is a concrete state of the STATE PATTERN of the resource square.

Concrete state when the resource square has no resource and the game mode is red.

It has a reference to the square.

  WithResourceState : Class

It is a concrete state of the STATE PATTERN of the resource square.

Concrete state when the resource square has at least a resource.

It has a reference to the square.

  NoResourceState : Class

It is a concrete state of the STATE PATTERN of the resource square.

Concrete state when the resource square has no resource and the game mode is green.

It has a reference to the square.

  IResourceSquareStateFactory : Class

Interface for all the ResourceSquareStateFactory. ResourceSquareStateFactory is responsible for the resource square state creation.

  RedResourceSquareStateFactory : Class

RedResourceSquareStateFactory is responsible for the resource square state creation if the game mode is red.

  GreenResourceSquareStateFactory : Class

GreenResourceSquareStateFactory is responsible for the resource square state creation if the game mode is green.

  MSurpriseSquare : Class

It is responsible for the creation of all the Surprise Tokens.

It knows the action to perform if the marker lands on it and how to setup the square.

It knows its type.

  ISurpriseTokenCommand : Class

Interface for all the surprise token to perform a generic operation.

COMMAND PATTERN

  SkipTurnSurpriseTokenCommand : Class

It is a concreteCommand component of the COMMAND PATTERN.

The ConcreteCommand defines a binding between an action and a Receiver.

It knows to who ask to perform the action and which is the action to perform.

Delegate the playerFactory for the creation of the player with the ability

to skip the next turn.

  GoToBuildingSiteSquareSurpriseTokenCommand : Class

It is a concreteCommand component of the COMMAND PATTERN.

The ConcreteCommand defines a binding between an action and a Receiver.

It knows to who ask to perform the action and which is the action to perform.

Delegate the player to land its marker on the building site square.

  TakeAnotherTurnSurpriseTokenCommand : Class

It is a concreteCommand component of the COMMAND PATTERN.

The ConcreteCommand defines a binding between an action and a Receiver.

It knows to who ask to perform the action and which is the action to perform.

Delegate the playerFactory for the creation of the player with the ability

to take another turn.

  TwoResourcesSupriseTokenCommand : Class

It is a concreteCommand component of the COMMAND PATTERN.

The ConcreteCommand defines a binding between an action and a Receiver.

It knows to who ask to perform the action and which is the action to perform.

Delegate the playerFactory for the creation of the player with the ability

to get 2 resources from the resource square.

  MINextHutTokenStrategy : Class

  MRandomNextHutTokenStrategy : Class

  MResourceSquareGreenSetupStrategy : Class
  FixedNumberResourceSquareRedSetupStrategy : Class

NumberResourceSquareSetupStrategy is a class responsible to setup a MResourceSquare

with a constant number (RESOURCENUMBER) of MResource.

RESOURCENUMBER is 2

MResource type is the same type of the resourceType of the square

This strategy implements the rule N.5 of the StoneAge Supplementary Specification

  MResourceSquareRedSetupStrategy : Class

MNumPlayersSquareSetupStrategy is a class responsible to setup a MResourceSquare

with a NumGamePlayers + ADDNUMBER of MResource.

ADDNUMBER is 1

MResource type is the same type of the resourceType of the square

This strategy implements the rule N.5 of the StoneAge Supplementary Specification

  MBuildingSiteSquare : Class

MBuildingSiteSquare is a persistence entity.

It is responsible for the creation of all hut tokens and it knows them and handle them until they are built.
