Model Elements
Name Description
  SkipNextTurnAbilityPlayerDecorator : Class

Define a player with the ability to skip the turn.

  Take2ResourcesAbilityPlayerDecorator : Class

Define a player with the ability to take 2 resource when the player is on a resource square.

  ExtraAbilityPlayerDecorator : Class

Hold a reference to a player with no ability and define a compliant interface to the player interface.

  TakeAnotherTurnAbilityPlayerDecorator : Class

Define a player with the ability to take another turn.

  MNoFindNewSquareStrategy : Class

MNoFindNewSquareStrategy is a class responsible to find the new square when the forest token is not of MDieFaceTokenForest or MSquareTokenForest type.

  MSquareTFFindNewSquareStrategy : Class

MSquareTFFindNewSquareStrategy is a class responsible to find the new square if the forest token value is a square name.

  MIFindNewSquareStrategy : Class

MIFindNewSquareStrategy is Strategy Interface to find the new square according to the forest token value.

  MFindNewSquareStrategyFactory : Class

Java class responsible for creating strategies to identifies the new square where move the marker.

With this class it is possible to dynamically modify the new square determination policy

by specifying a different Strategy class to be created.

MFindNewSquareStrategyFactory is a singleton, factory method class to create the right strategy.

  MDieFaceTFFindNewSquareStrategy : Class

MDieFaceTFFindNewSquareStrategy is a class responsible to find the new square if the forest token value is a number.

  PlayerAbilityCombiningStrategyConcreteFactory : Class

Factory to create the player ability combining strategy.

  IPlayerAbilityCombiningStrategyFactory : Class

Define a common interface to create the player ability combining strategy.

  SetOnlyNewAbilityStrategy : Class

This strategy set only the new ability to the player.

  KeepCurrentAbilityStrategy : Class

This strategy keeps only the current ability.

  IPlayerAbilityCombiningStrategy : Class

Define a common interface for the strategy to combining the player abilities.

  KeepBothAbilitiesStrategy : Class

This strategy keeps all the abilities.

  IPlayerFactory : Class

Interface for the player factory.

  MPlayerFactory : Class

PlayerFactory is responsible for the player creation.

  MIRobbedPlayerStrategy : Class

Define an interface to get the robbed player name.

  MRandomRobbedPlayerStrategy : Class

Strategy to choose randomly a robbed player.

  MEmulatedPlayer : Class

A specific player. The system emulates the human actions.

  MHumanPlayer : Class

A specific player.

  MMarker : Class

Persistent class.

It knows its name, in which square it is.

It is observable.

  MPlayer : Class

The class Player is responsible for the marker and settlement creation, marker movement, build a hut token, steal a resource, choose the name player to rob.

It knows if is the winner.

  MSettlement : Class

Java class to manage player's resources.

It is observable (by ResourceSquare and UPlayer).
